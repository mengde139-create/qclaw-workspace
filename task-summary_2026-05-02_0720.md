# 任务修复记录 2026-05-02

## 目标
修复首页任务栏全流程检查报告中识别的 Bug #1 ~ #6

## 修复内容

### Bug #1（严重）：submit.js 所有类型都调 evaluate/submit
- **根因**：早起/跑步/冥想/读书走文本评估会崩（缺content），早起打卡不应走AI评估
- **修复**：submit.js 的 submitWork() 中，写作类型调 `evaluate/submit`，其余类型调 `evaluate/checkin`
- **附带修复**：checkin 数据传递更完整（冥想传duration+notes，跑步传distance，早起传wakeupHour/Minute）

### Bug #2（严重）：submit.wxml 引用 displayCount 未定义
- **根因**：JS data 中只有 wordCount，wxml 用了 displayCount
- **修复**：wxml 中 `{{displayCount}}` → `{{wordCount}}{{targetUnit}}`，显示格式改为 `500字 / 500字`

### Bug #3（严重）：早起打卡提交时 goalId 为空
- **根因**：首页跳转 submit 不传 goalId，且早起从 goal 页创建后 switchTab 回首页不传递
- **修复**：submit.js onLoad 中，如果 options.goalId 为空，从 `goalId_{type}` 缓存和 goalsList 中补取

### Bug #4（中等）：冥想/跑步/读书专属提交页无防重复提交
- **修复**：三个提交页 onLoad 时调用 checkTodayCompleted()，已完成则弹窗提示并跳转结果页
- **逻辑**：检查 lastSubmitDate_{type} 和 goalsList 中的 todayCompleted + lastCompletedDate

### Bug #5（中等）：本地 completedDays 双重计算
- **根因**：云端 checkin 已 inc(1) completedDays，本地 saveRecord 又 +1
- **修复**：三个专属提交页的 saveXxxRecord() 方法接收 cloudSuccess 参数：
  - 云端成功 → 本地只设 todayCompleted=true，不 +1
  - 云端失败 → 本地才 +1（纯本地降级）

### Bug #6（中等）：goal 页面中转跳转浪费页面栈
- **根因**：goal.js 对 meditation/running/reading 用 navigateTo 跳专属页，goal 页留在栈中
- **修复**：改用 redirectTo 替代 navigateTo，goal 页被替换不再占页面栈
